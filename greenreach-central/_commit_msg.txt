EB#547 — F.A.Y.E. Phases 2-4: WRITE tools, trust tiers, intelligence loop, analytics

Phase 2 — WRITE Tools + Trust Tiers:
- admin-ops-agent.js: 9 new WRITE tools (create/acknowledge/resolve alerts,
  classify transactions, process refunds, send emails, update farm notes,
  save admin memory) + 5 analytics READ tools
- Trust tier system: AUTO (safe writes execute immediately), QUICK_CONFIRM,
  CONFIRM (describe impact + wait for yes), ADMIN (critical — explicit confirm)
- admin-assistant.js: Pending action confirmation flow — write tools that
  need confirmation are stored, admin sees confirm/cancel buttons, "yes"
  executes the action, "cancel" aborts. Works in both /chat and /chat/stream.
- Updated system prompt: removed "Phase 1 read-only" placeholder, added
  write tool safety instructions with tier descriptions.

Phase 3 — Proactive Intelligence:
- NEW services/faye-intelligence.js: Background loop (every 15 min) running
  5 anomaly detectors: payment failure rate, farm heartbeat staleness,
  order volume drops/spikes, accounting debit/credit imbalances,
  unclassified transaction backlog.
- Auto-creates admin_alerts with deduplication (2h cooldown per domain+title).
- Daily briefing email at 7 AM to ADMIN_BRIEFING_EMAIL via SES.
- Wired into server.js startup alongside nightly audit service.

Phase 4 — Analytics & Pattern Learning:

  growth rates), analyze_order_patterns (peak days, buyer concentration,
  basket size), analyze_farm_performance (revenue per farm, heartbeat
  reliability), get_anomaly_report, get_decision_log.
- Decision logging: all write tool executions recorded to faye_decision_log
  table for pattern analysis and audit trail.
- Migration 034: faye_decision_log table + unique index on
  accounting_classifications.transaction_id for ON CONFLICT support.
