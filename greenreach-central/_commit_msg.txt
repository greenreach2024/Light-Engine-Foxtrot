Fix wholesale buyer deactivate/reactivate 'Buyer not found' error

Root cause: deactivateBuyer() and reactivateBuyer() only checked the
in-memory buyersById Map. After server restarts, the GET /buyers list
falls back to the DB and shows buyers, but the deactivate/reactivate
handlers could not find them in memory.

- wholesaleMemoryStore.js: extract hydrateRowIntoMaps() helper, add
  new hydrateBuyerById() export that loads a single buyer from DB
  into the in-memory Maps
- admin-wholesale.js: add DB fallback to both /deactivate and
  /reactivate handlers — if in-memory lookup fails, hydrate from DB
  then retry
