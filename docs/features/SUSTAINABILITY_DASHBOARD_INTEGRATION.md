# Sustainability & ESG Dashboard - Integration Requirements

## Current Status: DEMO DATA ONLY ⚠️

The Sustainability & ESG Dashboard currently displays **demo/synthetic data** generated by the backend. While the UI looks great and the calculations are realistic, the dashboard is **not connected to actual farm operations** yet.

---

## What the Dashboard Currently Does

### 1. **API Endpoints** (`backend/sustainability_esg.py`)
The backend provides 8 REST endpoints:

- `GET /api/sustainability/energy/usage` - Energy consumption data
- `GET /api/sustainability/water/usage` - Water usage and recycling rates
- `GET /api/sustainability/nutrients` - Nutrient efficiency tracking
- `GET /api/sustainability/waste/tracking` - Waste diversion and composting
- `GET /api/sustainability/carbon-footprint` - Total CO2 emissions
- `GET /api/sustainability/esg-report` - Complete ESG scorecard
- `GET /api/sustainability/dashboard` - Real-time dashboard summary
- `GET /api/sustainability/trends` - Historical trend data for charts

### 2. **ESG Scoring Algorithm**
The system calculates a 0-100 ESG score based on:

- **Energy Efficiency (30 points)**: % renewable energy + efficiency
- **Water Efficiency (25 points)**: % water recycled
- **Nutrient Efficiency (20 points)**: Waste reduction rate
- **Waste Management (15 points)**: Composting/recycling rate
- **Carbon Footprint (10 points)**: Low emissions target (<50 kg/day)

### 3. **Demo Data Generation**
Currently generates 90 days of realistic data:
- Energy: 280-320 kWh/day (15-35% solar)
- Water: 800-1200 L/day (85-95% recycled)
- Nutrients: Base A, Base B, Cal-Mag usage (5-15% waste)
- Waste: Weekly organic/packaging waste tracking
- Carbon: Combined energy + water + transport emissions

## Integration Challenge: Missing Data Sources

### ❌ **What's NOT Being Measured**

#### 1. **Energy Monitoring**
- **Problem**: No actual energy meters or smart plugs measuring consumption
- **Required Hardware**:
  - Smart power meters on main circuits
  - Individual plug monitors for lights, HVAC, pumps
  - Solar panel production monitor (if applicable)
- **Data Needed**:
  - Real-time kWh usage per zone/device
  - Grid vs solar breakdown
  - Peak demand times
  - Cost per kWh from utility bills

#### 2. **Water Monitoring**
- **Problem**: No flow meters on water lines
- **Required Hardware**:
  - Flow meters on fresh water intake
  - Flow meters on recycled water loops
  - Tank level sensors
- **Data Needed**:
  - Total daily water consumption
  - Water recycled through system
  - Fresh water added
  - Evapotranspiration losses

#### 3. **Nutrient Usage Tracking**
- **Current**: Nutrient management page exists but not connected
- **Required Integration**:
  - Parse nutrient dosing logs from automation system
  - Track nutrient concentrate consumption
  - Measure waste from EC/pH adjustments
  - Link to inventory depletion rates
- **Data Needed**:
  - mL of each nutrient used per day
  - Waste from dumping/adjusting solution
  - Efficiency percentage (used vs wasted)

#### 4. **Waste Tracking**
- **Problem**: No automated waste measurement
- **Required Process**:
  - Manual weighing of organic waste (trimmings, culls)
  - Tracking of packaging materials
  - Composting/recycling bin weights
- **Data Needed**:
  - Weekly waste kg by category
  - Diversion rates (composted/recycled vs landfill)
  - Could be manual data entry initially

#### 5. **Carbon Footprint - Transport**
- **Integration Point**: ALREADY EXISTS in wholesale buyer portal!
- **Location**: `public/wholesale.js` - `loadEnvironmentalImpact()` method
- **Calculation**: Uses buyer location + farm location to calculate delivery distance
- **Formula**: `distance_km × 0.161 kg CO₂/km = carbon_per_delivery`
- **Issue**: **NOT connected to farm sustainability dashboard**

The **Wholesale Buyer Portal** (`/wholesale.html`) calculates environmental impact based on:

1. **Buyer Location** (e.g., Kingston, ON)
2. **Farm Locations** (fetched from `/api/farms/available`)
3. **Distance Calculation** (weighted average for multi-farm orders)
4. **Carbon Formula**: `0.161 kg CO₂ per km`
5. **Comparison**: Shows savings vs California produce baseline

## Integration Roadmap

### Phase 1: Nutrient Data (Easiest) ✅ Already Tracked

**Source**: Nutrient management system already records usage
**Action Required**:
1. Find where nutrient dosing is logged (automation system or manual records)
2. Create endpoint: `GET /api/nutrients/usage-log?days=30`
3. Return format:
   ```json
   {
     "date": "2025-12-23",
     "nutrient_type": "Base A",
     "volume_ml": 350,
     "waste_ml": 25,
     "efficiency_percent": 92.9
   }
   ```
4. Connect to sustainability dashboard API instead of demo data

### Phase 2: Transport Carbon (Already Calculated) ✅ Exists in Wholesale

**Action Required**:
1. Add endpoint: `GET /api/wholesale/carbon-summary?farm_id=GR-00001&days=30`
2. Aggregate all orders for the farm:
   - Total deliveries in period
   - Total kg CO₂ from transport
   - Average distance per delivery
3. Feed into sustainability dashboard `/carbon-footprint` endpoint
4. Update ESG carbon score to include transport

**Example Integration**:
```python
# In sustainability_esg.py
@router.get("/carbon-footprint")
async def get_carbon_footprint(days: int = 30):
    # Existing: energy + water carbon
    energy_carbon = calculate_energy_carbon(days)
    water_carbon = calculate_water_carbon(days)
    
    # NEW: Fetch transport carbon from wholesale orders
    transport_carbon = await fetch_transport_carbon(days)
    
    total_kg = energy_carbon + water_carbon + transport_carbon
    return {"total_kg": total_kg, ...}
```

### Phase 3: Energy Monitoring (Requires Hardware)

**Option A: Smart Plugs (Quick Start)**
- Install TP-Link Kasa or Tuya smart plugs on major equipment
- Use manufacturer APIs to pull kWh data
- Aggregate per hour/day
- Feed into `/api/sustainability/energy/usage` endpoint

**Option B: Professional Monitoring (Accurate)**
- Install Emporia Vue or Sense home energy monitor
- Monitors entire electrical panel
- Real-time circuit-level data
- API integration available

**Option C: Manual Utility Bills (Basic)**
- Enter monthly kWh from utility bills
- Divide by days to get daily average
- Better than nothing, but no real-time data

### Phase 4: Water Monitoring (Requires Hardware)

**Option A: Flow Meters**
- Install inline flow meters on main water lines
- Examples: YF-S201 (cheap) or professional ultrasonic meters
- Connect to ESP32/Arduino for data logging
- POST to backend: `/api/sustainability/water/usage`

**Option B: Manual Tank Tracking**
- Record reservoir fill times and volumes
- Estimate daily consumption from refill frequency
- Less accurate but zero hardware cost

### Phase 5: Waste Tracking (Manual Entry)

**Action Required**:
1. Create simple waste logging page in farm admin
2. Prompt weekly: "Weigh your waste bins"
3. Form inputs:
   - Organic waste (kg)
   - Packaging/recycling (kg)
   - General trash (kg)
4. Store in database
5. Feed into `/api/sustainability/waste/tracking`

## Data Integration Architecture

### Current (Demo Data)
```
sustainability_esg.py
  └─ generate_demo_data()
       └─ Creates 90 days synthetic data
       └─ Stored in memory (lists)
       └─ API endpoints return demo data
```

### Proposed (Real Data)
```
Data Sources
├─ Energy: Smart Plugs API
│   └─ Poll every 15 minutes
│   └─ Store: energy_readings table
│
├─ Water: Flow Meters
│   └─ ESP32 publishes to MQTT
│   └─ Store: water_readings table
│
├─ Nutrients: Automation Logs
│   └─ Parse dosing events
│   └─ Store: nutrient_usage table
│
├─ Waste: Manual Entry Form
│   └─ Weekly admin data entry
│   └─ Store: waste_logs table
│
└─ Transport: Wholesale Orders
    └─ Aggregate order deliveries
    └─ Calculate distance × carbon rate
    └─ Store: order_carbon_footprint table

Database
├─ energy_readings (timestamp, device_id, kwh, source_type, cost, carbon)
├─ water_readings (timestamp, zone_id, liters_used, recycled, efficiency)
├─ nutrient_usage (timestamp, nutrient_type, volume_ml, waste_ml)
├─ waste_logs (date, waste_type, weight_kg, recycled_kg, composted_kg)
└─ order_carbon_footprint (order_id, delivery_distance_km, carbon_kg)

Sustainability API
├─ Read from database tables (not demo lists)
├─ Aggregate daily/weekly/monthly
├─ Calculate ESG score from real metrics
└─ Return to dashboard UI
```

## Database Schema for Real Data

```sql
-- Energy tracking
CREATE TABLE energy_readings (
    id SERIAL PRIMARY KEY,
    timestamp TIMESTAMPTZ NOT NULL,
    device_id VARCHAR(50),
    device_name VARCHAR(100),
    kwh_used FLOAT NOT NULL,
    source_type VARCHAR(20) CHECK (source_type IN ('grid', 'solar', 'wind', 'hybrid')),
    cost_cad FLOAT,
    carbon_kg FLOAT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Water monitoring
CREATE TABLE water_readings (
    id SERIAL PRIMARY KEY,
    timestamp TIMESTAMPTZ NOT NULL,
    zone_id VARCHAR(50),
    liters_used FLOAT NOT NULL,
    recycled_liters FLOAT DEFAULT 0,
    efficiency_percent FLOAT,
    cost_cad FLOAT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Nutrient usage
CREATE TABLE nutrient_usage (
    id SERIAL PRIMARY KEY,
    timestamp TIMESTAMPTZ NOT NULL,
    nutrient_type VARCHAR(50) NOT NULL,
    volume_ml FLOAT NOT NULL,
    waste_ml FLOAT DEFAULT 0,
    efficiency_percent FLOAT,
    zone_id VARCHAR(50),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Waste tracking
CREATE TABLE waste_logs (
    id SERIAL PRIMARY KEY,
    log_date DATE NOT NULL,
    waste_type VARCHAR(20) CHECK (waste_type IN ('organic', 'packaging', 'general', 'hazardous')),
    weight_kg FLOAT NOT NULL,
    recycled_kg FLOAT DEFAULT 0,
    composted_kg FLOAT DEFAULT 0,
    landfill_kg FLOAT DEFAULT 0,
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Transport carbon from orders
CREATE TABLE order_carbon_footprint (
    id SERIAL PRIMARY KEY,
    order_id VARCHAR(50) UNIQUE NOT NULL,
    farm_id VARCHAR(50) NOT NULL,
    buyer_id VARCHAR(50) NOT NULL,
    delivery_date DATE,
    delivery_distance_km FLOAT NOT NULL,
    carbon_kg FLOAT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```
