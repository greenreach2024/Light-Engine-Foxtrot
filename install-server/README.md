# Light Engine Installation Server

**⚠️ DEPRECATED:** This server is no longer used. All installers are now hosted on AWS S3 + CloudFront for better performance and reliability.

**See:** [AWS_INSTALLER_HOSTING_SETUP.md](../AWS_INSTALLER_HOSTING_SETUP.md) for current deployment process.

---

## Legacy Overview (Historical Reference)

This server previously powered the one-line installer:
```bash
curl -sSL https://install.greenreach.io | bash
```

**Current Hosting:** AWS S3 bucket `light-engine-installers` with CloudFront CDN

## Endpoints

- `GET /` - Installation script (install.sh)
- `GET /greenreach-public.pem` - Public key for license verification
- `GET /lightengine-linux-x64` - Binary for Linux x86_64
- `GET /lightengine-linux-arm64` - Binary for Linux ARM64
- `GET /lightengine-{platform}.sha256` - SHA-256 checksum
- `GET /binaries` - List all available binaries
- `GET /stats` - Installation statistics
- `GET /health` - Health check

## Directory Structure

```
install-server/
├── server.js           # Express server
├── package.json        # Dependencies
├── README.md           # This file
├── binaries/           # Compiled binaries (generated by pkg)
│   ├── lightengine-linux-x64
│   ├── lightengine-linux-x64.sha256
│   ├── lightengine-linux-arm64
│   └── lightengine-linux-arm64.sha256
└── logs/               # Access logs
    └── access.log
```

## Setup

1. Install dependencies:
```bash
cd install-server
npm install
```

2. Build binaries (from project root):
```bash
npm run build:pkg
```

3. Start server:
```bash
npm start
```

## Development

Run with auto-reload:
```bash
npm run dev
```

## Production Deployment

### Option 1: AWS Elastic Beanstalk

1. Create `.ebextensions` for Node.js:
```yaml
option_settings:
  aws:elasticbeanstalk:container:nodejs:
    NodeCommand: "node server.js"
```

2. Deploy:
```bash
eb init
eb create install-greenreach
eb deploy
```

### Option 2: Docker

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
```

### Option 3: PM2

```bash
pm2 start server.js --name install-server
pm2 save
pm2 startup
```

## Security Considerations

- Binaries are served over HTTPS only
- SHA-256 checksums provided for integrity verification
- Public key served for RSA signature verification
- Rate limiting should be added in production
- CDN recommended for binary distribution

## Monitoring

Check server health:
```bash
curl https://install.greenreach.io/health
```

View installation statistics:
```bash
curl https://install.greenreach.io/stats
```

List available binaries:
```bash
curl https://install.greenreach.io/binaries
```

## Logs

Access logs stored in `logs/access.log`:
- All download requests
- IP addresses
- Timestamps
- Platform statistics

## Building Binaries

Binaries are created using `pkg` (see Todo #7). They will be automatically placed in `binaries/` directory and checksums generated.

From project root:
```bash
npm run build:pkg
```

This creates:
- `lightengine-linux-x64` - For x86_64 systems
- `lightengine-linux-arm64` - For ARM64 systems (Raspberry Pi, etc.)

## Testing

Test installation script locally:
```bash
# Start server
npm start

# In another terminal, test download
curl -sSL http://localhost:3000/ > /tmp/install.sh
bash /tmp/install.sh
```
